# 產品需求文件 (PRD) - 個人記帳簿

## 1. 專案概述
**背景與動機**
現代學生與上班族生活忙碌，往往難以持續追蹤個人財務狀況。許多現成的記帳軟體功能過於繁雜，或是充滿廣告，導致使用者很快放棄。因此，我們需要一套專注於「快速、簡單」的個人記帳系統，降低記帳門檻，幫助使用者輕鬆培養財務管理的習慣。

**目標用戶**
- 學生
- 上班族
- 剛開始學習理財，需要簡單記帳工具的人

**核心價值主張**
- **快**：極簡的操作流程，幾秒鐘即可完成一筆記帳。
- **簡單就可以用**：直覺的使用者介面，無需複雜的設定與學習。
- **免費**：提供核心的記帳服務完全免費。

---

## 2. 功能需求
系統將提供以下主要功能與使用者故事：

1. **快速記錄收入與支出**
   - 作為 **使用者**，我希望 **能快速輸入金額、選擇分類並送出儲存**，以便 **在消費當下能立刻記錄，不會忘記**。
2. **簡單的收支分類管理**
   - 作為 **使用者**，我希望 **有預設且常見的收支分類（如：飲食、交通、薪資等）供我點選**，以便 **簡化操作流程，無須自己思考分類**。
3. **首頁餘額統計顯示**
   - 作為 **使用者**，我希望 **能在登入首頁直接看到總收入、總支出與當前結餘的數字**，以便 **一句話掌握近期的財務健康狀況**。
4. **歷史紀錄查詢列表**
   - 作為 **使用者**，我希望 **能查看近期的收支流水帳明細**，以便 **回顧與確認每一筆金錢的去向**。
5. **免費輕量的圓餅圖表分析**
   - 作為 **使用者**，我希望 **能看到簡單的圓餅圖顯示本月各大分類的支出比例**，以便 **快速找出自己的主要花費來源並進行未來調整**。

*(註：使用者要求中的「快、簡單、免費」已轉化為具體的系統核心功能規劃與體驗訴求。)*

---

## 3. 非功能需求

**技術限制**
- 依據專案規範，需使用：
  - **後端框架**：Flask
  - **模板引擎**：Jinja2
  - **資料庫**：SQLite (非常適合輕量級個人應用)

**效能與安全考量**
- **效能**：頁面載入與表單送出速度需盡可能快（小於 1-2 秒），以符合「快」的核心價值。
- **安全**：基本的網頁安全防護（防止 SQL 注入、XSS 等），確保使用者的紀錄資料安全不會輕易佚失。

---

## 4. MVP (最小可行性產品) 範圍

**Must Have (必須有 - Phase 1)**
- 新增收入、新增支出紀錄功能。
- 在頁面列出所有收支明細。
- 計算並顯示總收入、總支出與總餘額。

**Should Have (應該有 - Phase 2)**
- 簡易的支出圓餅圖表。
- 可依據月份切換查看不同期間的紀錄。

**Nice to Have (可以有 - Phase 3)**
- 允許使用者自訂新增分類。
- 資料匯出（如 CSV 格式）功能。
